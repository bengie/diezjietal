Fonts & Texts

The Godfather of CSS Eric Meyer had the honour to kick-off the conference. Seeing Mr. Meyer in the flesh was all the convincing I needed to buy me a ticket. If you've ever heard a rumour of this being a man that rarely smiles, I have to disappoint you. It turned out to be a very kind, laughable and approachable guy.

Playing with fonts, also known as typography, doesn't have to be limited to including various webfonts but it can be CSS trickery as well.

Some examples

{% highlight css %} /* example 1 */ @font-face { font-family: "xkycd"; /* rename your long font, save some bytes */ src: url("Xylophone-Regular-Condensed.eot"); /* the font file */ src: local("Xylophone Regular condensed"), /* bulletproof, use it from the system. Could save a HTTP request */ ... } body { font: 1em/1.5 xkycd, sans-serif; } /* example 2 */ @font-face { font-stretch: expanded; /* this one I never heard of before */ font-family: Helvetica; /* name it Helvetica */ src: local('Impact'); /* but use Impact */ } /* example 3 */ @font-face { font-family: "Helvetica"; /* name it Helvetica */ font-weight: 100; /* and use it when font-weight is 100 */ src: local("Zapfino"); /* but use system font Zapfino */ } body { font: 1em Helvetica, sans-serif; } h1::first-letter { font-weight: 100; /* the trigger */ font-size: 200; } /* this results in a word with the first letter in a swirly font, while rest of word is in real Helvetica */ {% endhighlight %}

Did you ever have to create a fancy heading with a stylized & (e.g. Beauty & the beast)? Yes, you did. You would tackle this with an extra span surrounding the character like this <h1>Beauty <span>&</span> the beast</h1>. With unicode-range this is possible without the use of that extra unsemantic span.

{% highlight css %} @font-face { font-family: Helvetica; src: local("Zapfino"); /* use Zapfino */ unicode-range: U+26; /* will target ampersand (&) only */ } {% endhighlight%}

Disclaimer: Use with caution. Test in your browsers.

Flexbox

Stephen immediately busted the myth that flexbox is for page layout. It's for UI layout (e.g. navbars) while grid layout and page layout (these specs will merge) are meant for page layouts.

The talk was a fly-over of what's possible and how the syntax works.
